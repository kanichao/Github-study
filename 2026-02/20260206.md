### Google Colab、GitHub Codespaces、GitHub Actionsの違いは何？

<br>

**1. Google Colab：対話型実験環境** <br>

* 主な役割: プロトタイピング（試作）、データ解析、実験。 <br>
* 実行形態: 対話型 (Interactive)。セル単位でコードを実行し、即座に結果を確認。<br>
* 持続性: セッション単位。ブラウザを閉じたり一定時間放置したりすると、メモリ上のデータは消去される。ノートブック自体は Google ドライブに保存される。<br>
* 特徴: GUI（画面）の出力に強く、グラフの描画や turtle のような視覚的フィードバックが必要な学習に最適。

<br>

**2. GitHub Codespaces：統合開発環境** <br>

* 主な役割: 本格的なコーディング、プロジェクトの構造構築。<br>
* 実行形態: 常駐型 (Full-Stack)。ブラウザ上で動作する VS Code そのもの。ファイルシステム全体を管理。<br>
* 持続性: 準永続的。作業を中断しても、次回の起動時に「昨日の続き」から再開可能。<br>
* 特徴: 複数のファイルを跨ぐ複雑なプログラムの作成や、パッケージ管理、ディレクトリ構造の整理など、ソフトウェアエンジニアリングの実践に向き。

<br>

**3. GitHub Actions：自動実行環境** <br>

* 主な役割: 定期的なタスク、自動化、テストの実行（CI/CD）。<br>
* 実行形態: バッチ型 / 非対話型 (Headless)。あらかじめ定義された「ワークフロー（YAMLファイル）」に従って、裏側で勝手に動く。<br>
* 持続性: 非永続的 (Ephemeral)。タスクが完了した瞬間に、そのコンピュータは初期化（消去）される。<br>
* 特徴: 人間が操作する必要がないため、毎日決まった時間にスクリプトを実行する、あるいはコードが更新された時に自動でテストを行うといった「工場のライン」のような役割を果たします。<br>

<br>

### 実際の使い分けはどうする？

1. **Colab**でのプロトタイピング
   「特定の場所から文字を読み込み、書き換える」といったロジックが正しく動くか、一行ずつ検証する。 <br>

2. **Codespace**でのモジュール化と実装
   検証済みのロジックを独立した Python ファイル（.py）として整理し、リポジトリのディレクトリ構造を整える。<br>

3. **Actions**での自動化・デプロイ
   完成したプログラムを「毎日実行するタスク」として設定し、人間が手を動かさなくてもシステムが自律的に稼働するようにする。 <br>
